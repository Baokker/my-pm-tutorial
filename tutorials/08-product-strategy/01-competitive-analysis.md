# 01 - 竞品分析

> "知己知彼，百战不殆。" —— 《孙子兵法》

竞品分析（Competitive Analysis）是产品经理最核心的基本功之一。无论你是在做一款全新的产品，还是在迭代一款成熟产品，了解竞争对手在做什么、怎么做、为什么这么做，都是制定产品策略的前提。

---

## 一、为什么要做竞品分析？

### 1.1 竞品分析的价值

| 价值维度 | 具体说明 | 举例 |
|---------|---------|------|
| 了解市场格局 | 知道谁在做同样的事，市场有多大 | 了解在线文档市场有飞书、腾讯文档、石墨等 |
| 发现机会和差距 | 找到竞品没有满足的用户需求 | 发现竞品都不支持离线编辑 |
| 避免重复造轮子 | 竞品已经验证过的方案可以借鉴 | 竞品的某种交互设计已被用户接受 |
| 制定差异化策略 | 知道别人做什么，才能决定自己不做什么 | 别人做全功能，我专注做垂直场景 |
| 说服利益相关者 | 用竞品数据支撑你的产品决策 | 向老板证明某功能的市场需求存在 |
| 预判行业趋势 | 从竞品的动作推测行业发展方向 | 竞品纷纷接入 AI，说明 AI 是趋势 |

### 1.2 什么时候需要做竞品分析？

```
    ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐
    │  新产品立项  │  │ 大版本规划  │  │ 融资/汇报  │  │ 定期追踪   │
    │  阶段       │  │  阶段      │  │  准备      │  │（月度/季度）│
    └──────┬─────┘  └──────┬─────┘  └──────┬─────┘  └──────┬─────┘
           │               │               │               │
           ▼               ▼               ▼               ▼
     全面深度分析      聚焦功能对比     市场格局梳理     增量动态追踪
     (2-4 周)        (1-2 周)        (1-2 周)         (持续进行)
```

## 二、如何识别竞品

不是所有"看起来类似"的产品都是你的竞品。我们需要区分三种不同类型：

### 2.1 三种竞品类型

```
    ┌─────────────────────────────────────────────────────────┐
    │                     竞争关系图谱                         │
    │                                                         │
    │              ┌─────────────────────┐                    │
    │              │     替代品           │                    │
    │              │   (不同方式解决      │                    │
    │              │    同一需求)         │                    │
    │              │   ┌─────────────┐   │                    │
    │              │   │  间接竞品    │   │                    │
    │              │   │ (部分重叠)   │   │                    │
    │              │   │ ┌─────────┐ │   │                    │
    │              │   │ │ 直接竞品 │ │   │                    │
    │              │   │ │(核心竞争)│ │   │                    │
    │              │   │ └─────────┘ │   │                    │
    │              │   └─────────────┘   │                    │
    │              └─────────────────────┘                    │
    │                                                         │
    └─────────────────────────────────────────────────────────┘
```

| 类型 | 定义 | 以"微信读书"为例 |
|-----|------|----------------|
| **直接竞品** | 目标用户相同、核心功能相同、商业模式相似 | 网易蜗牛读书、Kindle App、多看阅读 |
| **间接竞品** | 目标用户有重叠、部分功能相似、但定位不同 | 得到 App（偏知识付费）、樊登读书（偏听书） |
| **替代品** | 用完全不同的方式满足相同的底层需求（获取知识） | 播客、短视频知识类内容、线下书店 |

### 2.2 如何找到竞品？

| 方法 | 操作方式 | 适用场景 |
|-----|---------|---------|
| 应用商店搜索 | 在 App Store / Google Play 搜索关键词 | 移动端产品 |
| 搜索引擎搜索 | Google / 百度搜索"XX类产品""XX替代品" | 所有类型 |
| 行业报告 | 查看艾瑞、QuestMobile、Gartner 等报告 | 了解市场格局 |
| 投融资信息 | IT桔子、天眼查查看同赛道融资情况 | 了解新入场的竞争者 |
| 用户访谈 | 直接问用户"你还用什么类似产品？" | 发现用户真实的替代选择 |
| 社交媒体 | 微博、知乎、小红书搜索用户讨论 | 了解用户口碑 |
| 产品社区 | Product Hunt、少数派、V2EX | 发现新锐产品 |

## 三、竞品分析的维度

一份完整的竞品分析，应该从多个维度进行对比：

### 3.1 功能对比

功能对比是最基础也最直观的维度。建议用表格形式呈现：

| 功能模块 | 我方产品 | 竞品 A | 竞品 B | 竞品 C |
|---------|---------|--------|--------|--------|
| 核心功能 1 | 支持 | 支持 | 部分支持 | 不支持 |
| 核心功能 2 | 支持 | 不支持 | 支持 | 支持 |
| 差异化功能 | 支持 | 不支持 | 不支持 | 不支持 |
| ... | ... | ... | ... | ... |

> **小技巧**：功能对比不要只看"有没有"，还要看"做得好不好"。可以用评分制（1-5 分）或者标注体验细节。

### 3.2 用户体验

| 体验维度 | 评估方法 |
|---------|---------|
| 信息架构 | 画出产品的功能结构图，对比层级深度 |
| 交互设计 | 核心任务的操作步骤数对比 |
| 视觉设计 | UI 风格、品牌调性、设计一致性 |
| 性能体验 | 启动速度、加载速度、流畅度 |
| 新手引导 | 首次使用体验、上手难度 |

### 3.3 商业模式

| 对比项 | 需要关注的问题 |
|--------|---------------|
| 盈利方式 | 靠什么赚钱？广告？订阅？交易佣金？ |
| 定价策略 | 价格区间？免费版和付费版的区别？ |
| 获客渠道 | 主要通过什么渠道获取用户？ |
| 变现效率 | ARPU（每用户平均收入）大约是多少？ |

### 3.4 技术架构（简要了解）

| 维度 | 观察方法 |
|-----|---------|
| 技术栈 | 通过招聘 JD、技术博客、开源项目了解 |
| 性能表现 | 实际使用体验 + 第三方监测工具 |
| 开放能力 | 是否有 API、插件系统、开发者生态 |
| 数据安全 | 数据存储方式、隐私政策、合规情况 |

### 3.5 市场份额与增长趋势

| 数据来源 | 说明 |
|---------|------|
| QuestMobile | 中国移动互联网用户数据 |
| App Annie / data.ai | 全球应用市场数据 |
| SimilarWeb | 网站流量数据 |
| 财报数据 | 上市公司公开的用户和收入数据 |
| 第三方行业报告 | 艾瑞、易观、Gartner 等 |

## 四、SWOT 分析详解

SWOT 是竞品分析中最经典、最常用的框架之一。它帮助你从四个维度全面评估一个产品（或公司）的竞争态势。

### 4.1 SWOT 四象限

```
    ┌─────────────────────────────┬─────────────────────────────┐
    │                             │                             │
    │    S - Strengths (优势)      │    W - Weaknesses (劣势)     │
    │                             │                             │
    │    内部的、有利的因素         │    内部的、不利的因素         │
    │                             │                             │
    │    - 核心技术壁垒            │    - 团队经验不足            │
    │    - 品牌认知度高            │    - 资金有限               │
    │    - 用户基数大              │    - 产品功能单一            │
    │    - 数据积累丰富            │    - 技术债务重             │
    │                             │                             │
    ├──────────────── 内部因素 ────┼──────────────────────────────┤
    │                             │                             │
    │    O - Opportunities (机会)  │    T - Threats (威胁)        │
    │                             │                             │
    │    外部的、有利的因素         │    外部的、不利的因素         │
    │                             │                             │
    │    - 新技术趋势（AI）        │    - 巨头入场               │
    │    - 政策利好               │    - 监管趋严               │
    │    - 新市场空间              │    - 用户需求变化            │
    │    - 竞品犯错的机会          │    - 替代品出现             │
    │                             │                             │
    └─────────────── 外部因素 ────┴──────────────────────────────┘

                      有利因素              不利因素
```

### 4.2 SWOT 的交叉分析

单纯列出 SWOT 四个象限只完成了一半工作。更有价值的是做**交叉分析**，形成策略：

| 策略类型 | 交叉组合 | 含义 | 举例 |
|---------|---------|------|------|
| **SO 策略（进攻）** | 优势 + 机会 | 利用优势抓住机会 | 利用 AI 技术优势抢占智能化赛道 |
| **WO 策略（扭转）** | 劣势 + 机会 | 借助机会弥补劣势 | 借助行业风口获取融资扩大团队 |
| **ST 策略（防御）** | 优势 + 威胁 | 利用优势应对威胁 | 利用品牌壁垒应对新进入者 |
| **WT 策略（撤退）** | 劣势 + 威胁 | 最小化劣势和威胁 | 收缩业务线，聚焦核心场景 |

## 五、波特五力分析

迈克尔·波特（Michael Porter）提出的五力模型是分析行业竞争强度的经典框架。

### 5.1 五力模型图

```
                          ┌─────────────────┐
                          │   潜在进入者      │
                          │  的威胁           │
                          │                  │
                          │  (进入壁垒高低？) │
                          └────────┬─────────┘
                                   │
                                   ▼
    ┌──────────────┐     ┌─────────────────┐     ┌──────────────┐
    │  供应商的      │     │                 │     │  购买者的     │
    │  议价能力      │────▶│  现有竞争者之间  │◀────│  议价能力     │
    │              │     │  的竞争          │     │              │
    │ (上游依赖度？) │     │                 │     │ (用户选择多？)│
    └──────────────┘     │  (竞争激烈程度)   │     └──────────────┘
                         └────────┬─────────┘
                                  │
                                  ▼
                         ┌─────────────────┐
                         │  替代品的威胁     │
                         │                  │
                         │  (其他解决方案？) │
                         └──────────────────┘
```

### 5.2 五力分析详解

| 力量 | 核心问题 | 分析要点 |
|-----|---------|---------|
| **现有竞争者** | 行业内竞争有多激烈？ | 竞争者数量、行业增速、产品差异化程度、退出壁垒 |
| **潜在进入者** | 新玩家容易进来吗？ | 资金门槛、技术门槛、品牌壁垒、政策壁垒、网络效应 |
| **替代品** | 用户会转向其他方案吗？ | 替代品的性价比、转换成本、用户习惯 |
| **供应商议价** | 上游对你有多大话语权？ | 供应商集中度、替换成本、对核心资源的依赖程度 |
| **买家议价** | 用户有多大选择权？ | 用户数量、转换成本、价格敏感度、信息透明度 |

### 5.3 以「在线会议」行业为例

| 五力维度 | 分析结果 | 强度 |
|---------|---------|------|
| 现有竞争者 | 腾讯会议、钉钉、飞书、Zoom 等激烈竞争 | 强 |
| 潜在进入者 | 技术门槛中等，但需要大量服务器投入 | 中 |
| 替代品 | 电话、微信语音/视频通话、面对面会议 | 中 |
| 供应商议价 | 云服务商（AWS、阿里云）有一定议价权 | 中 |
| 买家议价 | 企业客户集中采购，议价能力强 | 强 |

## 六、竞品分析报告模板结构

一份专业的竞品分析报告通常包含以下结构：

```
    竞品分析报告结构

    1. 分析背景与目的
       ├── 为什么做这次分析
       ├── 分析范围和深度
       └── 关键问题定义

    2. 市场概况
       ├── 市场规模与增长率
       ├── 主要玩家分布
       └── 行业趋势

    3. 竞品选择与概览
       ├── 竞品选择理由
       ├── 各竞品基本信息（成立时间、融资、团队）
       └── 产品定位对比

    4. 详细对比分析
       ├── 功能对比（表格）
       ├── 用户体验对比
       ├── 商业模式对比
       ├── 技术架构对比
       └── 运营策略对比

    5. SWOT 分析
       ├── 各竞品的 SWOT
       └── 我方产品的 SWOT

    6. 结论与建议
       ├── 核心发现
       ├── 差异化方向
       └── 具体行动建议

    7. 附录
       ├── 数据来源
       └── 详细数据表格
```

## 七、竞品分析实操：笔记类产品对比

下面我们以笔记类产品为例，对 **Notion**、**Obsidian** 和 **语雀** 进行实操分析。

### 7.1 产品概览

| 维度 | Notion | Obsidian | 语雀 |
|-----|--------|----------|------|
| 所属公司 | Notion Labs（美国） | Dynalist（加拿大） | 蚂蚁集团（中国） |
| 成立时间 | 2016 年 | 2020 年 | 2018 年 |
| 产品定位 | All-in-one 工作空间 | 本地优先的知识管理 | 专业的知识管理与协作平台 |
| 目标用户 | 个人用户 + 团队 | 个人知识管理者、研究者 | 企业团队 + 个人 |
| 平台覆盖 | Web + Desktop + Mobile | Desktop + Mobile（第三方同步） | Web + Desktop + Mobile |
| 估值/融资 | 估值约 100 亿美元 | 未公开融资 | 蚂蚁集团旗下 |

### 7.2 功能对比

| 功能 | Notion | Obsidian | 语雀 |
|-----|--------|----------|------|
| 富文本编辑 | 优秀（块编辑器） | 基础 Markdown | 优秀（类 Word） |
| 双向链接 | 支持 | 核心功能（强） | 部分支持 |
| 知识图谱 | 不支持 | 核心功能（强） | 不支持 |
| 数据库/表格 | 核心功能（强） | 需插件 | 支持（表格、看板） |
| 团队协作 | 核心功能（强） | 弱（需第三方） | 核心功能（强） |
| 离线使用 | 部分支持 | 完全支持（本地优先） | 部分支持 |
| 插件/扩展 | 有限 | 丰富（社区生态） | 有限 |
| AI 功能 | Notion AI（付费） | 需插件 | AI 助手 |
| 中文支持 | 一般 | 一般 | 优秀 |
| 数据导出 | Markdown/CSV | 原生 Markdown | Markdown/PDF |

### 7.3 用户体验对比

| 体验维度 | Notion | Obsidian | 语雀 |
|---------|--------|----------|------|
| 上手难度 | 中等（概念多） | 较高（需学 Markdown） | 低（所见即所得） |
| 编辑流畅度 | 一般（依赖网络） | 优秀（本地处理） | 良好 |
| 信息组织 | 灵活（页面嵌套） | 灵活（文件夹+链接） | 清晰（知识库结构） |
| 美观程度 | 高（设计精良） | 中等（可定制主题） | 良好 |
| 搜索功能 | 良好 | 优秀（即时搜索） | 良好 |

### 7.4 商业模式对比

| 维度 | Notion | Obsidian | 语雀 |
|-----|--------|----------|------|
| 盈利模式 | Freemium + 团队订阅 | 付费功能（同步/发布） | Freemium + 企业版 |
| 个人免费版 | 有（功能受限） | 核心功能完全免费 | 有（空间受限） |
| 付费价格 | $8-15/人/月 | $4-8/月（可选） | 会员 99-199 元/年 |
| 企业版 | 支持 | 不支持 | 支持 |
| 收入来源 | 主要靠团队订阅 | 个人付费功能 | 企业客户 + 个人会员 |

### 7.5 SWOT 对比分析

**Notion 的 SWOT：**

| 优势 (S) | 劣势 (W) |
|---------|---------|
| All-in-one 理念深入人心 | 性能问题（大文档卡顿） |
| 品牌影响力强 | 数据安全顾虑（云端存储） |
| 投资充裕、团队强大 | 中文本地化不够好 |
| 数据库功能强大 | 学习曲线较陡 |

| 机会 (O) | 威胁 (T) |
|---------|---------|
| AI 赋能提升产品价值 | 竞品不断追赶 |
| 企业市场空间大 | 用户对数据隐私越来越敏感 |
| 全球化扩张 | 经济下行企业缩减预算 |

**Obsidian 的 SWOT：**

| 优势 (S) | 劣势 (W) |
|---------|---------|
| 本地优先，数据安全 | 没有原生团队协作 |
| 插件生态丰富 | 对非技术用户不友好 |
| 双向链接和知识图谱强 | 没有商业化融资 |
| 社区活跃度高 | 移动端体验待提升 |

| 机会 (O) | 威胁 (T) |
|---------|---------|
| 隐私意识增强利好本地方案 | Notion 等大厂持续投入 |
| PKM（个人知识管理）趋势 | AI 时代笔记形态可能被颠覆 |
| 学术和研究领域需求 | 开源竞品（Logseq）竞争 |

**语雀的 SWOT：**

| 优势 (S) | 劣势 (W) |
|---------|---------|
| 背靠蚂蚁集团资源 | 品牌认知度不如 Notion |
| 中文体验优秀 | 国际化能力弱 |
| 企业协作能力强 | 开放性不足 |
| 文档+知识库结构清晰 | 用户社区不够活跃 |

| 机会 (O) | 威胁 (T) |
|---------|---------|
| 国产替代需求 | 飞书文档等强竞争 |
| 企业数字化转型 | 母公司策略变化风险 |
| AI 技术应用空间大 | 免费策略可能影响收入 |

### 7.6 核心结论

```
    三款产品的差异化定位

    功能丰富度
    ▲
    │  ★ Notion
    │     "瑞士军刀"
    │        All-in-one
    │
    │                    ★ 语雀
    │                      "团队知识库"
    │                       协作优先
    │
    │                                 ★ Obsidian
    │                                   "思考者的工具"
    │                                    本地 + 深度
    └──────────────────────────────────────────────▶
                                          个人化/自主性
```

| 产品 | 一句话总结 | 最适合的用户 |
|-----|----------|------------|
| Notion | 功能最全面的工作空间，适合需要统一工具的团队 | 初创团队、自由职业者 |
| Obsidian | 最尊重数据所有权的知识管理工具 | 研究者、写作者、程序员 |
| 语雀 | 最适合中文用户的企业知识管理平台 | 国内企业团队、技术团队 |

## 八、竞品追踪：如何持续跟踪竞品动态

竞品分析不是一次性的工作，而是需要持续进行的。

### 8.1 建立竞品追踪体系

    ┌──────────────────────────────────────────────────┐

    │  │ 版本更新  │   │          │    │ 产品规划  │     │

    │  │ 收集     │   │          │    │          │     │

### 8.2 追踪维度与频率

| 追踪维度 | 频率 | 方法 |
|---------|------|------|
| 产品版本更新 | 每周 | 关注竞品的更新日志、App Store 更新记录 |
| 功能变化 | 每两周 | 实际使用竞品，体验新功能 |
| 运营活动 | 每周 | 关注竞品的官方社交媒体、公众号 |
| 人才招聘 | 每月 | 关注竞品在招聘网站发布的岗位（能看出战略方向） |
| 融资/合作 | 实时 | 设置关键词 Google Alert 或使用IT桔子 |
| 用户口碑 | 每月 | 监控应用商店评分、社交媒体舆情 |
| 财报数据 | 每季度 | 上市公司的财报发布 |

## 九、竞品分析常见误区

| 误区 | 正确做法 |
|-----|---------|
| 只看功能，不看战略 | 要理解竞品为什么这样做，背后的逻辑是什么 |
| 分析了不行动 | 每次分析都应该有明确的结论和行动建议 |
| 只关注直接竞品 | 替代品和潜在进入者同样重要 |
| 一次分析就完事 | 竞品分析需要持续追踪和更新 |
| 主观臆断替代数据 | 尽量用数据说话，减少个人偏见 |
| 照搬竞品功能 | 要思考差异化，而不是简单跟随 |
