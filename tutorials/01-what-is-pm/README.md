## 章节结构

### 01 - 产品经理的角色定义与职责

> 文件：[01-pm-role.md](./01-pm-role.md)

本节将回答最根本的问题：**产品经理是谁？做什么？** 你将看到：

- 产品经理的一句话定义与深度拆解

- PM 与其他容易混淆角色的横向对比
- PM 在团队中的定位与桥梁角色

### 02 - 产品经理的分类

> 文件：[02-pm-types.md](./02-pm-types.md)

产品经理并不是一种笼统的岗位。不同的业务方向对 PM 的要求天差地别。本节将带你认识：

- C 端产品经理（面向消费者）
- B 端产品经理（面向企业 / SaaS）
- 数据产品经理
- AI 产品经理
- 增长产品经理（Growth PM）
- 平台型产品经理
- 各类 PM 的核心差异对比

### 03 - 职业发展路径与能力模型

> 文件：[03-pm-career.md](./03-pm-career.md)

了解了角色和分类之后，最重要的问题就是：**我该怎么成为一个 PM？成为 PM 之后又该怎么成长？** 本节将覆盖：

## 核心概念预览

在正式开始之前，先快速预览几个核心概念：

| 概念 | 一句话解释 |
|------|------------|
| 产品经理（PM） | 负责定义「做什么产品」以及「为什么做」的人 |
| 需求 | 用户在特定场景下想要解决的问题或达成的目标 |
| MVP | 最小可行产品，用最少的资源验证核心假设 |
| 用户画像 | 对目标用户群体的典型特征描述 |
| PRD | 产品需求文档，描述产品功能的详细规格 |
| OKR | 目标与关键成果，衡量产品目标达成情况 |
| DAU/MAU | 日活/月活跃用户数，衡量产品健康度的基础指标 |
