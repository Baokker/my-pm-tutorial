# 第三章：用户研究与需求分析

> "不了解用户的产品经理，就像不了解病人的医生。" —— 产品管理箴言

---

## 模块概览

用户研究是产品经理最核心的基本功之一。一个好的产品不是"拍脑袋"想出来的，
而是建立在对用户深刻理解的基础上。本章将系统性地带你学习用户研究的全流程：
从理解用户是谁（用户画像），到如何科学地收集用户声音（调研方法），
再到如何将海量信息提炼为可执行的需求（需求分析），
最后用可视化的方式呈现用户的完整体验（旅程地图）。

## 章节目录

### [01 - 用户画像（Persona）](./01-user-persona.md)
用户画像是产品设计的起点。你将学习如何基于真实数据构建典型用户的"虚拟代表"，
让团队在讨论产品时能对齐"我们在为谁做产品"。

### [02 - 用户调研方法](./02-user-research-methods.md)
掌握定性和定量两大类调研方法，包括用户访谈、问卷调查、可用性测试等，
学会根据不同阶段选择合适的调研手段。

### [03 - 需求收集与优先级排序](./03-requirement-analysis.md)
需求永远做不完，资源永远不够用。学习如何系统地收集需求，
并使用 KANO、RICE、MoSCoW 等经典框架来决定"先做什么"。

### [04 - 用户旅程地图 & 故事地图](./04-user-journey.md)
从用户的视角出发，将他们与产品交互的全过程可视化，
发现体验断点和改进机会，让产品设计更有全局观。

## 学习建议

1. **先通读再实践**：建议先把 4 个小节快速过一遍，建立整体认知，然后再逐个深入练习。
2. **找一个真实场景练手**：选一个你常用的 App（如外卖、社交、购物），尝试用本章的方法分析它。
3. **输出倒逼输入**：每学完一节，尝试输出一份对应的文档（画像文档/访谈提纲/需求优先级表/旅程地图）。
4. **不要追求完美**：第一次做用户研究难免生疏，重要的是建立思维框架，技巧会随经验提升。
