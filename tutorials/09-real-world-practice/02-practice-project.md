## 项目背景

### 虚拟项目：「校园二手交易平台」-- 校淘

**项目简介：** 为在校大学生打造的二手物品交易平台，解决毕业季/换宿舍时大量闲置物品的处置问题，以及新生入学时低价购买学习和生活用品的需求。

**为什么选择这个项目？**
- 场景真实、易于理解
- 用户群体明确（大学生）
- 涉及双边市场的产品逻辑
- 包含社交 + 交易 + 信任体系等多个产品维度

```
  +---------------------------------------------------------------+
  |                                                               |
  |   毕业生/高年级                       新生/低年级               |
  |   +-----------+                      +-----------+            |
  |   |  闲置物品 |   ====校淘====>      |  低价购买  |            |
  |   |  需要处理 |   <====校淘====      |  学习/生活 |            |
  |   +-----------+                      |  用品      |            |
  |                                      +-----------+            |
  |                                                               |
  +---------------------------------------------------------------+
```

## 第一步：市场调研 & 竞品分析

### 1.1 市场分析

**市场规模估算（自上而下）：**

```
  全国在校大学生数量:        ~4500 万人
  有闲置物品处置需求的比例:   ~60%  => 2700 万人
  愿意使用线上平台交易的比例: ~50%  => 1350 万人（潜在用户）
  年人均交易额:              ~300 元
  潜在市场规模（TAM）:       ~40 亿元/年

  +----------------------------+
  |  TAM: 40 亿元              |
  |  +----------------------+  |
  |  |  SAM: 15 亿元        |  |
  |  |  +----------------+  |  |
  |  |  |  SOM: 2 亿元   |  |  |
  |  |  +----------------+  |  |
  |  +----------------------+  |
  +----------------------------+

  TAM = 全部潜在市场
  SAM = 可服务市场（头部 50 所高校）
  SOM = 可获得市场（首年目标 5 所高校）
```

### 1.2 竞品分析

| 竞品 | 定位 | 优势 | 劣势 | 与我们的差异 |
|------|------|------|------|-------------|
| **闲鱼** | 全品类二手交易 | 用户基数大、支付宝信用体系 | 非校园场景、信息庞杂 | 我们聚焦校园，更精准 |
| **转转** | 二手 3C 交易 | 品类专业、验机服务 | 品类有限、非校园 | 我们覆盖全品类校园物品 |
| **校园 BBS / 跳蚤群** | 校内信息发布 | 免费、校内信任度高 | 无交易保障、信息混乱 | 我们提供结构化交易流程 |
| **小红书二手** | 种草 + 二手 | 内容丰富、社区氛围好 | 非专业交易平台 | 我们专注交易闭环 |

**竞品分析矩阵图：**

```
  交易保障 (高)
       ^
       |
       |   转转           校淘(目标位置)
       |   *               *
       |
       |           闲鱼
       |            *
       |
       |
       |   校园BBS        小红书
       |    *               *
       +-------------------------------> 校园场景适配度 (高)
  (低)
```

## 第三步：需求列表 & 优先级排序（RICE 评分）

### RICE 评分说明

| 维度 | 含义 | 评分范围 |
|------|------|---------|
| **R**each（触达） | 这个功能影响多少用户？ | 1-10（10=影响所有用户） |
| **I**mpact（影响） | 对用户体验的提升程度？ | 1-5（5=极大提升） |
| **C**onfidence（信心） | 对这个评估有多大把握？ | 50%-100% |
| **E**ffort（工作量） | 需要多少人/周的开发量？ | 1-10（10=工作量极大） |

**RICE 分数 = (R x I x C) / E**

## 第六步：PRD 核心章节

### 6.1 用户故事

| 编号 | 角色 | 用户故事 | 验收标准 |
|------|------|---------|---------|
| US-001 | 卖家 | 作为卖家，我希望能快速发布闲置物品，以便尽快找到买家 | 3 步内完成发布；支持多图上传；必填项：标题、图片、价格、分类 |
| US-002 | 买家 | 作为买家，我希望能按学校/分类浏览商品，以便找到我需要的物品 | 支持按学校、分类、价格区间筛选；支持关键词搜索 |
| US-003 | 买家 | 作为买家，我希望能查看卖家的信用评分，以便判断是否可信 | 展示信用评分、历史交易数、用户评价 |
| US-004 | 买卖双方 | 作为用户，我希望能与对方即时沟通，以便协商价格和交易细节 | 支持文字/图片消息；未读消息提醒；可发送商品卡片 |
| US-005 | 买卖双方 | 作为用户，我希望能预约面交时间和地点，以便线下完成交易 | 支持选择时间/地点；双方确认机制；交易前提醒 |
| US-006 | 用户 | 作为用户，我希望用学校邮箱认证身份，以便确保交易对象都是真实在校学生 | 支持 .edu 邮箱验证；认证状态可见；未认证用户有功能限制 |

### 6.2 功能描述：发布商品

**功能名称：** 发布闲置商品

**功能描述：** 卖家可以通过拍照或从相册选择图片，填写商品信息后一键发布闲置物品。

**详细规则：**

| 字段 | 必填 | 规则 |
|------|------|------|
| 商品图片 | 是 | 至少 1 张，最多 9 张；支持拍照和相册选择；单张 <10MB |
| 标题 | 是 | 2-30 字符；不允许特殊符号 |
| 描述 | 否 | 最多 500 字符 |
| 分类 | 是 | 从预设分类中选择（二级分类） |
| 成色 | 是 | 全新/几乎全新/轻微使用痕迹/明显使用痕迹 |
| 价格 | 是 | 0.01-99999 元；支持两位小数 |
| 原价 | 否 | 用于展示折扣力度 |
| 面交地点 | 是 | 默认当前校区；可选择具体地点 |

**异常情况处理：**

| 异常情况 | 处理方式 |
|---------|---------|
| 图片上传失败 | 提示"上传失败，请重试"，保留已上传图片 |
| 内容包含违规信息 | 机审拦截 + 人工复核，提示"内容审核中" |
| 网络中断 | 自动保存草稿，恢复网络后提示继续编辑 |
| 重复发布 | 检测相似商品，提示"您似乎已发布过类似商品" |

## 第七步：数据指标定义

### 7.1 北极星指标

```
  +---------------------------------------------------------------+
  |                                                               |
  |  北极星指标 (North Star Metric)                                |
  |                                                               |
  |  +=========================================================+  |
  |  ||                                                       ||  |
  |  ||    月度完成交易数 (Monthly Completed Transactions)      ||  |
  |  ||                                                       ||  |
  |  +=========================================================+  |
  |                                                               |
  |  选择理由:                                                     |
  |  1. 直接衡量平台核心价值（撮合交易）                             |
  |  2. 同时反映买家和卖家的活跃度                                  |
  |  3. 是商业化的基础（未来可在交易中抽佣）                         |
  |  4. 可拆解为多个可执行的子指标                                  |
  |                                                               |
  +---------------------------------------------------------------+
```

### 7.2 AARRR 指标体系

```
  +---------------------------------------------------------------+
  |                     AARRR 海盗指标                              |
  +---------------------------------------------------------------+
  |                                                               |
  |  Acquisition (获客)                                            |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |  |  新增注册用户数                |                              |
  |  |  注册转化率                    |                              |
  |  |  各渠道获客成本 (CAC)          |                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |                  |                                             |
  |                  v                                             |
  |  Activation (激活)                                             |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |  |  完成学生认证率                |                              |
  |  |  首次发布/首次浏览率           |                              |
  |  |  新用户首日留存率              |                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |                  |                                             |
  |                  v                                             |
  |  Retention (留存)                                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |  |  次日/7日/30日留存率           |                              |
  |  |  周活跃卖家数                  |                              |
  |  |  周活跃买家数                  |                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |                  |                                             |
  |                  v                                             |
  |  Revenue (收入)                                                |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |  |  月交易总额 (GMV)             |                              |
  |  |  平均客单价                    |                              |
  |  |  付费增值服务收入              |                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |                  |                                             |
  |                  v                                             |
  |  Referral (推荐)                                               |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |  |  邀请注册转化率                |                              |
  |  |  分享次数/人均                 |                              |
  |  |  NPS (净推荐值)               |                              |
  |  +--+--+--+--+--+--+--+--+--+--+                              |
  |                                                               |
  +---------------------------------------------------------------+
```

### 7.3 核心指标看板

| 指标类别 | 指标名称 | 定义 | V1.0 目标 |
|---------|---------|------|----------|
| **获客** | 日新增注册数 | 当日完成注册的用户数 | 100+ |
| **获客** | 注册转化率 | 注册数 / 访问数 | >30% |
| **激活** | 认证完成率 | 完成学生认证 / 注册用户 | >60% |
| **激活** | 首次发布率 | 首次发布商品 / 认证用户 | >20% |
| **留存** | 次日留存率 | 次日登录用户 / 当日新用户 | >40% |
| **留存** | 7 日留存率 | 7 日内登录 / 当日新用户 | >20% |
| **交易** | 日均完成交易数 | 每日确认完成的交易 | 10+ |
| **交易** | 发布到成交时长 | 商品发布到完成交易的中位数 | <72h |
| **体验** | 聊天响应时长 | 买家首次发消息到卖家回复 | <2h |
| **体验** | 交易完成率 | 完成交易 / 发起聊天 | >30% |
