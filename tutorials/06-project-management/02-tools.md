## 一、项目管理工具

### 1.1 Jira

**Atlassian 出品，全球最流行的项目管理工具之一。**

#### 核心功能

```
  ┌─────────────────── Jira 核心功能 ─────────────────────┐
  │                                                       │
  │  ┌───────────┐  ┌───────────┐  ┌───────────┐        │
  │  │  看板/面板 │  │  Backlog  │  │  Sprint   │        │
  │  │  Board     │  │  管理     │  │  规划     │        │
  │  └───────────┘  └───────────┘  └───────────┘        │
  │                                                       │
  │  ┌───────────┐  ┌───────────┐  ┌───────────┐        │
  │  │  Issue     │  │  工作流   │  │  报表/    │        │
  │  │  跟踪     │  │  自定义   │  │  仪表盘   │        │
  │  └───────────┘  └───────────┘  └───────────┘        │
  │                                                       │
  │  ┌───────────┐  ┌───────────┐  ┌───────────┐        │
  │  │  Roadmap  │  │  自动化   │  │  插件     │        │
  │  │  路线图   │  │  规则     │  │  生态     │        │
  │  └───────────┘  └───────────┘  └───────────┘        │
  └───────────────────────────────────────────────────────┘
```

#### Jira 的 Issue 类型

| Issue 类型 | 用途 | 示例 |
|-----------|------|------|
| **Epic** | 大的功能模块/主题 | "用户注册登录系统" |
| **Story** | 用户故事，具体需求 | "用户可以通过手机号注册" |
| **Task** | 具体任务 | "设计注册页面原型" |
| **Sub-task** | 任务的子任务 | "编写手机号校验逻辑" |
| **Bug** | 缺陷/问题 | "注册时验证码无法发送" |

#### Jira 工作流示例

```
  Jira 默认工作流

  ┌────────┐     ┌────────────┐     ┌────────┐
  │  待办   │────►│   进行中    │────►│  完成   │
  │ To Do  │     │In Progress │     │  Done  │
  └────────┘     └────────────┘     └────────┘
       ▲                                 │
       └─────────── 重新打开 ────────────┘

  自定义工作流（更常见）

  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
  │ 待办 │─►│ 开发 │─►│ 开发 │─►│ 测试 │─►│ 测试 │─►│ 已上 │
  │      │  │ 中   │  │ 完成 │  │ 中   │  │ 通过 │  │ 线   │
  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘
```

#### Jira 适用场景

- 中大型团队（20人以上）
- 需要复杂工作流自定义
- 需要与 Confluence、Bitbucket 等 Atlassian 工具集成
- 企业级权限管理需求

#### Jira 优缺点

| 优点 | 缺点 |
|------|------|
| 功能极其强大，几乎能满足所有需求 | 学习曲线陡峭，新手上手困难 |
| 高度可定制，工作流、字段可自由配置 | 界面较为复杂，操作不够直观 |
| 丰富的插件生态（Marketplace） | 加载速度较慢（尤其是云端版） |
| 强大的搜索和筛选（JQL） | 配置过度会导致团队"工具疲劳" |
| 完善的权限管理 | 价格较贵（按人头收费） |

### 1.2 Linear

**近年来最受欢迎的新一代项目管理工具，以"快"著称。**

```
  ┌──────────────── Linear 设计哲学 ────────────────┐
  │                                                  │
  │    "Build software faster"                       │
  │                                                  │
  │    ┌──────────────────────────────────────────┐  │
  │    │  极简设计  +  极致性能  +  键盘优先      │  │
  │    └──────────────────────────────────────────┘  │
  │                                                  │
  │    与 Jira 的对比：                               │
  │                                                  │
  │    Jira:    功能全面 ──────────── 复杂            │
  │    Linear:  够用就好 ──────────── 简洁            │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

#### Linear 核心特点

| 特点 | 说明 |
|------|------|
| **极致速度** | 全键盘操作，毫秒级响应，本地优先渲染 |
| **简洁设计** | 没有过度的配置项，开箱即用 |
| **Cycle 机制** | 类似 Sprint 但更轻量灵活 |
| **自动化** | 内置自动化规则（如自动分配、状态变更） |
| **Git 集成** | 与 GitHub/GitLab 深度集成，PR 自动关联 Issue |
| **Roadmap** | 内置产品路线图功能 |

#### 为什么 Linear 越来越流行？

1. **开发者喜欢**：键盘快捷键丰富，操作流畅
2. **设计美观**：暗色主题，现代化 UI
3. **学习成本低**：比 Jira 简单很多
4. **适合小团队**：不需要复杂配置就能用得很好
5. **与开发工作流深度集成**：自动同步 PR 和 Issue 状态

#### Linear 适用场景

- 小到中型团队（5-50 人）
- 技术驱动的团队
- 追求效率和简洁的团队
- SaaS 产品开发团队

#### Trello 特点

| 优点 | 缺点 |
|------|------|
| 极其简单，5 分钟上手 | 功能有限，不适合复杂项目 |
| 免费版功能够用 | 缺少高级报表和分析 |
| 看板直观，拖拽操作 | 卡片多了不好管理 |
| 丰富的 Power-Up 扩展 | 企业级功能需要付费 |

#### Trello 适用场景

- 个人任务管理
- 小型团队（2-10 人）
- 非技术团队（运营、市场）
- 简单的项目/活动管理

### 1.4 Asana

**注重任务管理和团队协作的工具。**

#### Asana 核心特点

| 特点 | 说明 |
|------|------|
| **多视图** | 列表视图、看板视图、时间轴（甘特图）、日历视图 |
| **项目模板** | 内置丰富模板，快速启动 |
| **目标管理** | 支持 OKR / Goals 功能 |
| **工作负载** | 可视化团队成员工作量 |
| **跨项目引用** | 同一任务可关联到多个项目 |

#### Asana 适用场景

- 中型团队（10-100 人）
- 非纯技术团队
- 需要甘特图/时间轴功能
- 跨部门协作较多的团队

### 1.5 项目管理工具对比总结

| 维度 | Jira | Linear | Trello | Asana |

| **团队规模** | 中大型 | 小中型 | 小型 | 中型 |
| **学习成本** | 高 | 低 | 极低 | 中 |
| **功能深度** | 极深 | 中等 | 浅 | 中等 |
| **界面美观** | 一般 | 优秀 | 好 | 好 |
| **性能速度** | 较慢 | 极快 | 快 | 快 |
| **自定义** | 极强 | 中等 | 弱 | 中等 |
| **Scrum 支持** | 完善 | 支持 | 需扩展 | 基础 |
| **价格** | 较贵 | 中等 | 基础免费 | 基础免费 |
| **适合谁** | 大团队/复杂项目 | 技术团队 | 个人/小团队 | 跨职能团队 |
| **国内可用性** | 可用（偶尔卡） | 可用 | 可用 | 可用 |

## 二、文档协作工具

### 2.1 Notion

**"All-in-one" 工作空间，集文档、数据库、看板、Wiki 于一体。**

```
  ┌──────────────── Notion 核心功能 ────────────────┐
  │                                                  │
  │  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
  │  │  文档     │  │  数据库   │  │  Wiki    │      │
  │  │  Pages    │  │ Database │  │ 知识库   │      │
  │  └──────────┘  └──────────┘  └──────────┘      │
  │                                                  │
  │  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
  │  │  看板     │  │  日历     │  │  模板    │      │
  │  │  Board    │  │ Calendar │  │ Template │      │
  │  └──────────┘  └──────────┘  └──────────┘      │
  │                                                  │
  │  一切皆 Block，自由组合                           │
  └──────────────────────────────────────────────────┘
```

#### PM 在 Notion 中的典型用法

| 用途 | 说明 |
|------|------|
| **PRD 撰写** | 用 Page 写产品需求文档 |
| **需求池管理** | 用 Database 建立需求池，支持多种视图 |
| **会议记录** | 用模板记录每次会议 |
| **竞品分析** | 用表格整理竞品信息 |
| **团队 Wiki** | 建立产品知识库 |
| **OKR 管理** | 跟踪团队目标 |

#### Notion 优缺点

| 优点 | 缺点 |
|------|------|
| 功能全面，一个工具解决多个问题 | 国内访问速度一般 |
| 界面美观，使用体验好 | 功能太多，容易过度设计 |
| Block 化设计，灵活度极高 | 搜索功能有待加强 |
| 模板丰富 | 权限管理不够精细 |
| 支持 AI 辅助写作 | 离线能力较弱 |

### 2.2 Confluence

**Atlassian 出品的企业级知识库，常与 Jira 配合使用。**

#### Confluence 特点

| 特点 | 说明 |
|------|------|
| **与 Jira 深度集成** | 可以在文档中嵌入 Jira Issue |
| **空间管理** | 按团队/项目创建独立空间 |
| **权限管理** | 精细的访问权限控制 |
| **模板系统** | 内置 PRD、会议记录、技术方案等模板 |
| **版本管理** | 完整的文档版本历史 |

#### 适用场景

- 已经在用 Jira 的团队（天然集成）
- 企业级知识管理需求
- 需要严格权限控制的团队
- 技术文档密集的团队

---

### 2.3 飞书文档 / 语雀

**国内常用的文档协作平台。**

#### 飞书文档

```
  ┌──────────── 飞书文档生态 ────────────────┐
  │                                          │
  │  飞书 IM（沟通）                          │
  │     │                                    │
  │     ├── 飞书文档（协同编辑）              │
  │     ├── 飞书多维表格（数据库）            │
  │     ├── 飞书知识库（Wiki）                │
  │     ├── 飞书项目（项目管理，原飞书 Meego）│
  │     └── 飞书会议（视频会议）              │
  │                                          │
  │  一站式办公平台，国内中大型公司常用        │
  └──────────────────────────────────────────┘
```

| 优点 | 缺点 |
|------|------|
| 国内访问快，体验流畅 | 生态绑定飞书体系 |
| 多人实时协作体验好 | 导出能力有限 |
| 与飞书 IM 无缝集成 | 个人用户吸引力有限 |
| 多维表格功能强大 | 国际化团队不适用 |

#### 语雀

- 蚂蚁集团出品的知识管理工具
- 擅长结构化文档（知识库、技术文档）
- 支持画板、思维导图等多种内容类型
- 个人版免费，企业版付费

---

### 2.4 文档协作工具对比

| 维度 | Notion | Confluence | 飞书文档 | 语雀 |
|------|--------|-----------|---------|------|
| **定位** | All-in-one | 企业知识库 | 一站式办公 | 知识管理 |

| **灵活度** | 极高 | 中 | 高 | 中 |

| **免费额度** | 有 | 有限 | 有 | 有 |
| **适合谁** | 灵活小团队 | Jira 用户 | 飞书体系企业 | 技术团队 |

## 三、设计协作工具

### 3.1 Figma

**全球最主流的 UI 设计与协作工具。**

```
  ┌──────────────── Figma 工作流 ────────────────────┐
  │                                                   │
  │  设计师                 PM / 工程师                │
  │                                                   │
  │  ┌──────────┐          ┌──────────┐              │
  │  │ 在 Figma │  ──────► │ 在 Figma │              │
  │  │ 中设计   │  分享链接 │ 中查看   │              │
  │  └──────────┘          └────┬─────┘              │
  │                              │                    │
  │                         ┌────▼─────┐             │
  │                         │ 标注尺寸  │             │
  │                         │ 查看样式  │             │
  │                         │ 导出切图  │             │
  │                         │ 添加评论  │             │
  │                         └──────────┘             │
  │                                                   │
  │  核心优势：                                       │
  │  - 浏览器即可使用，无需安装                        │
  │  - 多人实时协作编辑                                │
  │  - 强大的组件系统（Components）                    │
  │  - 开发模式（Dev Mode）查看标注                   │
  │  - 原型交互（Prototyping）                        │
  └───────────────────────────────────────────────────┘
```

#### PM 需要掌握的 Figma 技能

| 技能 | 说明 |
|------|------|
| **查看设计稿** | 学会浏览 Figma 文件、切换页面 |
| **添加评论** | 在设计稿上直接留下反馈 |
| **查看标注** | 了解间距、字体、颜色等参数 |
| **播放原型** | 查看交互效果，验证流程 |
| **基础编辑** | 修改文案、调整简单布局（加分项） |

> **提示**：PM 不需要像设计师一样精通 Figma，但至少要会看和评论。

---

### 3.2 蓝湖

**国内最主流的设计交付与协作平台。**

#### 蓝湖的核心功能

| 功能 | 说明 |
|------|------|
| **设计稿管理** | 上传 Sketch/Figma/PS 设计稿 |
| **自动标注** | 自动生成标注信息，前端直接查看 |
| **切图下载** | 一键下载各种分辨率的切图 |
| **原型演示** | 支持简单的页面跳转原型 |
| **产品文档** | 可以直接在蓝湖中写 PRD |

#### Figma vs 蓝湖

| 维度 | Figma | 蓝湖 |
|------|-------|------|
| **定位** | 设计工具 + 协作 | 设计交付平台 |
| **设计能力** | 强（原生设计工具） | 弱（主要是查看） |
| **标注查看** | 内置 Dev Mode | 自动生成标注 |
| **国内体验** | 可用，但偶有卡顿 | 流畅 |
| **价格** | 免费版有限制 | 免费版够用 |
| **趋势** | 全球主流，越来越强 | 国内主流，但 Figma 在蚕食市场 |

| 工具 | 优点 | 缺点 | 适合 |
|------|------|------|------|
| **Slack** | 频道管理强、集成丰富、搜索强大 | 国内访问需科学上网 | 外企、国际团队 |
| **飞书** | 体验好、文档集成强、多维表格 | 生态封闭 | 互联网公司 |
| **钉钉** | 功能全面、OA 审批完善 | 过于臃肿，"已读"功能争议大 | 中小企业 |
| **企业微信** | 连接个人微信、客户管理 | 功能相对简单 | 需要触达 C 端用户的企业 |

### 4.2 沟通工具的使用技巧

作为 PM，高效使用沟通工具很重要：

| 技巧 | 说明 |
|------|------|
| **善用频道/群组** | 按项目建群，避免信息混杂 |
| **善用 @提及** | 确保相关人能看到重要信息 |
| **善用 Pin/置顶** | 重要决定、文档链接置顶 |
| **异步沟通** | 非紧急事项不要要求即时回复 |
| **减少群聊** | 能私聊解决的不要群聊 |
| **沟通留痕** | 重要决定在群里文字确认，不要只口头说 |

## 五、工具选择建议

### 5.1 按团队规模选择

| 团队规模 | 项目管理 | 文档协作 | 设计协作 | 沟通工具 |
|----------|---------|---------|---------|---------|
| **1-5 人（初创）** | Trello / Linear | Notion | Figma | Slack / 飞书 |
| **5-20 人（小团队）** | Linear / Jira | Notion / 飞书文档 | Figma / 蓝湖 | Slack / 飞书 |
| **20-100 人（中型）** | Jira / Linear | Confluence / 飞书文档 | Figma + 蓝湖 | 飞书 / 钉钉 |
| **100+ 人（大型）** | Jira | Confluence | Figma + 蓝湖 | 飞书 / 钉钉 / 企业微信 |

### 5.2 按场景选择

```
  ┌──────────────── 场景 → 工具推荐 ──────────────────────┐
  │                                                        │
  │  外企 / 国际团队                                       │
  │  └── Jira + Confluence + Figma + Slack                 │
  │                                                        │
  │  国内互联网公司（中大型）                               │
  │  └── Jira / 飞书项目 + 飞书文档 + Figma/蓝湖 + 飞书    │
  │                                                        │
  │  国内互联网创业公司                                     │
  │  └── Linear / Trello + Notion + Figma + 飞书           │
  │                                                        │
  │  传统企业数字化转型                                     │
  │  └── Jira / 钉钉项目 + 钉钉文档 + 蓝湖 + 钉钉/企微    │
  │                                                        │
  │  个人 / 自由职业                                       │
  │  └── Trello / Notion + Notion + Figma + 微信           │
  │                                                        │
  └────────────────────────────────────────────────────────┘
```

### 5.3 选工具的原则

| 原则 | 说明 |
|------|------|
| **团队已有 > 个人喜好** | 别想着换掉全公司的工具，先适应 |
| **简单够用 > 功能强大** | 不要为了用高级功能而增加学习成本 |
| **工具统一 > 各自为政** | 团队用同一套工具，减少信息孤岛 |
| **先免费 > 后付费** | 先用免费版验证是否适合，再考虑付费 |
| **数据可迁移** | 考虑未来切换工具的成本 |
