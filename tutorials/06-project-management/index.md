# 第六章：项目管理

## 模块概览

项目管理是产品经理的**核心硬技能**之一。很多人误以为产品经理只需要"想需求"、"画原型"，但现实中，一个需求从构思到最终上线，中间需要经历大量的协调、跟进、推动工作。**如果说产品设计是"做正确的事"，那么项目管理就是"把事做成"。**

## 为什么 PM 需要掌握项目管理能力？

### 1. 产品经理 ≠ 项目经理，但必须懂项目管理

在很多公司（尤其是中小型团队），并没有专职的项目经理（Project Manager）。产品经理往往需要**身兼数职**，既要定义需求，也要推动落地。即使在有专职项目经理的大公司，产品经理也需要理解项目管理的基本逻辑，才能更好地配合。

### 2. 想法再好，落不了地等于零

产品经理日常会面对这些场景：

| 场景 | 需要的项目管理能力 |
|------|-------------------|
| 需求排不上开发计划 | 优先级管理、资源协调 |
| 开发进度延期 | 风险识别、进度跟踪 |
| 上线后出现严重 Bug | 应急响应、复盘机制 |
| 多个需求并行推进 | 多项目管理、时间管理 |
| 跨部门协作推不动 | 沟通技巧、向上管理 |
| 需求频繁变更 | 变更管理、敏捷迭代 |

### 3. 项目管理能力是晋升的关键

初级 PM 可能只需要写好 PRD，但中高级 PM 必须具备独立推动复杂项目落地的能力。在面试中，"你是如何推动一个项目从 0 到 1 上线的？"几乎是必考题。

```
  PM 职业成长路径与项目管理的关系

  初级 PM ──────► 中级 PM ──────► 高级 PM ──────► 产品总监
     │               │               │               │
     │               │               │               │
  写好 PRD       独立带项目      管理多项目线     战略规划
  跟进需求       协调跨团队      建立流程规范     组织管理
     │               │               │               │
     ▼               ▼               ▼               ▼
  项目管理        项目管理         项目管理       项目管理
  基础能力        核心能力         高阶能力       战略能力
```

### 项目管理 vs 产品管理 的区别

很多新人会混淆"项目管理"和"产品管理"。以下表格帮你厘清：

| 维度 | 项目管理 | 产品管理 |
|------|---------|---------|
| **核心问题** | 怎么做？什么时候做完？ | 做什么？为什么做？ |
| **关注点** | 进度、资源、风险 | 用户需求、商业价值 |
| **时间跨度** | 项目周期（有明确开始和结束） | 产品生命周期（持续演进） |
| **成功标准** | 按时、按质、按预算交付 | 用户满意、业务增长 |
| **技能侧重** | 计划、协调、跟进 | 洞察、决策、设计 |
| **典型角色** | 项目经理（PM/PMO） | 产品经理（PM） |

> 在互联网公司，产品经理经常需要同时承担两种角色的职责。

## 一个需求从想法到上线的完整旅程

理解项目管理，最好的方式是看一个需求完整的生命周期：

```
  一个需求的完整旅程

  阶段1: 发现需求
  ┌──────────────────────────────┐
  │  用户反馈 / 数据分析 / 竞品  │
  │  老板指示 / 运营诉求          │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段2: 需求分析与设计
  ┌──────────────────────────────┐
  │  用户调研 → 需求定义 → PRD   │
  │  交互设计 → UI 设计          │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段3: 项目规划                     <── 项目管理开始介入
  ┌──────────────────────────────┐
  │  需求评审 → 技术评审          │
  │  任务拆分 → 工时估算 → 排期   │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段4: 开发执行
  ┌──────────────────────────────┐
  │  Sprint 启动 → 每日站会       │
  │  开发 → 联调 → Code Review   │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段5: 测试验收
  ┌──────────────────────────────┐
  │  冒烟测试 → 功能测试          │
  │  回归测试 → Bug 修复          │
  │  PM 验收                     │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段6: 上线发布
  ┌──────────────────────────────┐
  │  灰度发布 → 全量上线          │
  │  监控数据 → 线上巡检          │
  └──────────────┬───────────────┘
                 │
                 ▼
  阶段7: 复盘优化
  ┌──────────────────────────────┐
  │  数据复盘 → 用户反馈收集      │
  │  Sprint 回顾 → 迭代优化       │
  └──────────────────────────────┘
```

在这个旅程中，阶段 3 到阶段 7 都是项目管理的范畴。产品经理需要确保每个环节都顺利推进。

## 学习建议

1. **先学方法论，再学工具**：工具只是手段，理解敏捷思维才是根本
2. **动手实践**：注册 Jira / Trello 等工具的免费版，亲自体验
3. **关注软技能**：项目管理中，沟通能力往往比工具使用更重要
4. **复盘总结**：每次项目结束后，养成复盘的习惯
5. **从小项目开始**：不要一开始就挑战复杂项目，先从一个小功能的完整闭环练起
6. **多观察**：看看你身边优秀的 PM 是怎么开评审会、怎么跟进进度、怎么处理冲突的
